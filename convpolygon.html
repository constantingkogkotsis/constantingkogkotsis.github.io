Traversing a convex polygon.

An integrated project page based on a presentation for the course Introduction to Algorithm Design and Analysis. The problem asks for a shortest route from the westernmost to the easternmost vertex of a convex polygon while visiting every vertex.

The key point is that convexity turns an apparently factorial search over possible orders into a structured dynamic-programming problem over two ordered chains.

Problem statement

The original problem is written as a story about Dilcue, Otalp, and a convex-polygon lake. The mathematical core is a shortest Hamiltonian path problem with fixed endpoints, restricted by convex geometry.

English formulation

Translation

The famous geometer and nature-lover Dilcue builds a lake whose shape is a convex polygon with \(n\) vertices. At every vertex of the lake lies a beautiful theorem of his friend Otalp, together with an elegant “Proof from The Book.” A boat can be rented at the westernmost vertex \(s\) and must be returned at the easternmost vertex \(t\). The goal is to visit all \(n\) vertices while minimizing the total distance travelled. Since the boat moves with constant speed, this is equivalent to minimizing the total path length.

Input

The coordinates of the \(n\) vertices of a convex polygon. We assume, for simplicity, that no two vertices have the same \(x\)- or \(y\)-coordinate.

Output

An ordering of the vertices that starts at \(s\), ends at \(t\), visits every vertex, and minimizes the total Euclidean length.

Target

Design an \(O(n^2)\) algorithm. The intended method is dynamic programming, after extracting the geometric structure of the optimal path.

Geometric structure

Two geometric facts reduce the search space. First, a shortest valid path has no self-intersections. Second, on each side of the line \(st\), the vertices must be visited from west to east.

1. A shortest path does not cross itself

Uncrossing

If two non-adjacent edges \(ab\) and \(cd\) cross internally, we can replace them by \(ac\) and \(bd\). By the triangle inequality in the crossing quadrilateral, the new pair is shorter:

\[ |ab| + |cd| > |ac| + |bd|. \]

Therefore a crossing path cannot be optimal.

2. Vertices on the same side are ordered

Monotonicity

Let \(\ell = st\). The line \(\ell\) separates the remaining vertices into an upper chain \(U=(u_1,\ldots,u_a)\) and a lower chain \(L=(v_1,\ldots,v_b)\), each sorted by increasing \(x\)-coordinate.

In an optimal path, vertices from the same chain appear in this west-to-east order. Otherwise the path makes a backward move on one side of \(\ell\), forcing a crossing inside the convex polygon, contradicting the uncrossing fact.

\[ U=(u_1,\ldots,u_a), \quad L=(v_1,\ldots,v_b), \quad a+b=n-2. \]

Once the order inside each chain is fixed, the only remaining choice is the interleaving: at each step do we take the next upper vertex or the next lower vertex?

Dynamic programming

The state records how many upper-chain and lower-chain vertices have already been taken, plus the side on which the partial path currently ends.

States

Subproblems

Define \(u_0=v_0=s\). For \(0\le i\le a\) and \(0\le j\le b\):

\[ M_U(i,j)=\text{best partial path ending at }u_i, \]

\[ M_L(i,j)=\text{best partial path ending at }v_j. \]

In state \((i,j)\), the path has already visited \(u_1,\ldots,u_i\) and \(v_1,\ldots,v_j\). The only information we must remember is the endpoint of the partial path.

Recurrences

To end at \(u_i\), the previous vertex is either \(u_{i-1}\), continuing on the upper chain, or \(v_j\), switching from the lower chain.

\[ M_U(i,j)= \min\{M_U(i-1,j)+d(u_{i-1},u_i),\, M_L(i-1,j)+d(v_j,u_i)\}. \]

Similarly, to end at \(v_j\), the previous vertex is either \(u_i\) or \(v_{j-1}\).

\[ M_L(i,j)= \min\{M_U(i,j-1)+d(u_i,v_j),\, M_L(i,j-1)+d(v_{j-1},v_j)\}. \]

The initial condition is \(M_U(0,0)=M_L(0,0)=0\), with impossible states set to \(+\infty\).

Algorithm

The dynamic program fills states in increasing order of \(i+j\). This ensures that every state depends only on states already computed.

Shortest path through the vertices

Pseudocode
  1. Let \(s\) be the westernmost vertex and \(t\) the easternmost vertex.
  2. Split \(V(P)\setminus\{s,t\}\) into the upper chain \(U\) and lower chain \(L\) with respect to the line \(st\).
  3. Sort \(U=(u_1,\ldots,u_a)\) and \(L=(v_1,\ldots,v_b)\) by increasing \(x\)-coordinate.
  4. Set \(u_0\leftarrow s\), \(v_0\leftarrow s\), and initialize \(M_U,M_L\) with \(+\infty\).
  5. Set \(M_U(0,0)\leftarrow 0\) and \(M_L(0,0)\leftarrow 0\).
  6. For \(r=1,\ldots,a+b\):
  7. For all \((i,j)\) with \(0\le i\le a\), \(0\le j\le b\), and \(i+j=r\):
  8. If \(i\ge 1\), update \(M_U(i,j)\) from \(u_{i-1}\) or \(v_j\).
  9. If \(j\ge 1\), update \(M_L(i,j)\) from \(u_i\) or \(v_{j-1}\).
  10. Return \(\min\{M_U(a,b)+d(u_a,t),\,M_L(a,b)+d(v_b,t)\}\).

Final value and reconstruction

Backtracking

Once all vertices have been visited, the partial path ends either at the last upper-chain vertex \(u_a\) or the last lower-chain vertex \(v_b\). We then add the final edge to \(t\):

\[ \operatorname{OPT}= \min\{M_U(a,b)+d(u_a,t),\,M_L(a,b)+d(v_b,t)\}. \]

To recover the actual visiting order, store with every minimum the choice that attained it. Backtracking from the final choice to \((0,0)\) gives the path.

Correctness and complexity

The proof is a standard dynamic-programming induction once the geometric structure has reduced the problem to interleaving two ordered chains.

Correctness invariant

Induction

For every state \((i,j)\), the values \(M_U(i,j)\) and \(M_L(i,j)\) are the minimum lengths of all valid partial paths that have visited exactly \(u_1,\ldots,u_i\) and \(v_1,\ldots,v_j\), and end respectively at \(u_i\) or \(v_j\).

  • Base: \((0,0)\) is the trivial path at \(s\).
  • Step: the last move to \(u_i\) is from \(u_{i-1}\) or \(v_j\); the last move to \(v_j\) is from \(u_i\) or \(v_{j-1}\).
  • Conclusion: the recurrences check exactly all possible last moves, so every state is optimal.

Complexity

O(n²)

There are \((a+1)(b+1)\) states in each table. Since \(a+b=n-2\), this is \(O(n^2)\). Each state is computed in constant time. Sorting gives \(O(n\log n)\), so the total running time is \(O(n^2)\).

Final picture

Geometry + DP

Final message: the geometry reveals the structure of the shape; dynamic programming then builds the optimal route.
